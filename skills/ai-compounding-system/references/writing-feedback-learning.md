# Writing Feedback Learning

This file stores the user's writing preferences and draft revision lessons. Keep it local and update it after the user gives feedback on drafts.

## Read Before Writing

Use this before writing:

- articles,
- first drafts,
- cloud-document expansion drafts,
- public drafts,
- internal knowledge notes,
- presentation scripts.

## Default Writing Checks

- Start with the audience's concrete pain, not with a method name.
- State the main judgment early.
- Show Before/After when possible.
- Separate draft, polished draft, asset-library copy, and published version.
- Keep a bottom evidence/material index for drafts that may be reused later.
- Ask whether the user wants to continue from first draft to polished draft or public draft.
- Do not claim something is final, published, or archived unless it has been approved and executed.
